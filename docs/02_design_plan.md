# 02. 디자인 기획서

## 디자인 방향

NANO ERP는 반복 업무에 쓰이는 도구이므로 차분하고 밀도 있는 업무형 UI를 지향합니다. 첫 화면은 마케팅 랜딩이 아니라 실제 작업 대시보드가 되어야 합니다.

## 주요 화면

### 1. 로그인

- Google 로그인 버튼 중심의 단순한 화면
- 서비스명과 짧은 설명만 제공
- 로그인 실패 시 명확한 오류 메시지 표시

### 2. 대시보드

- 오늘 일정
- 오늘 할 일
- 최근 스캔 문서
- 최근 변환 보고서
- 빠른 업로드 버튼

### 3. 캘린더 / 투두

- 좌측: 캘린더 또는 날짜 목록
- 우측: 선택 날짜의 일정과 할 일
- 일정과 할 일을 구분하되 같은 업무 흐름에서 확인 가능하게 구성

### 4. 연락처

- 연락처 리스트
- 검색 및 필터
- 명함 스캔 업로드
- OCR 결과 확인 후 저장하는 검토 화면

### 5. 문서 스캔

- 문서 이미지 업로드
- 스캔본 리스트
- 문서명, 날짜, 태그, 원본 파일 정보 표시

### 6. 녹음 / 보고서

- 녹음파일 업로드
- 변환 상태 표시
- 변환 텍스트 미리보기
- 보고서 리스트와 상세 보기

## UI 원칙

- 업무 도구답게 과도한 장식보다 정보 밀도와 명확성을 우선한다.
- 반복되는 항목은 카드 또는 테이블로 구성한다.
- 주요 액션은 버튼에 아이콘과 텍스트를 함께 사용한다.
- 업로드, 변환, 저장 같은 비동기 작업은 상태를 분명히 보여준다.
- 모바일에서도 주요 목록과 업로드 동선이 깨지지 않게 한다.
