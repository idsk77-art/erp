# 03. 기능 기획서

## 1. 구글 로그인

### 요구사항

- Google OAuth로 로그인한다.
- 로그인한 사용자의 기본 프로필을 저장한다.
- 인증되지 않은 사용자는 보호된 화면에 접근할 수 없다.

### 주요 데이터

- 사용자 ID
- 이메일
- 이름
- 프로필 이미지
- 마지막 로그인 시각

## 2. 캘린더 / 투두리스트 관리

### 요구사항

- 일정을 생성, 조회, 수정, 삭제한다.
- 할 일을 생성, 조회, 수정, 완료, 삭제한다.
- 날짜 기준으로 일정과 할 일을 함께 확인한다.

### 주요 데이터

- 일정: 제목, 설명, 시작일시, 종료일시, 위치
- 할 일: 제목, 설명, 마감일, 완료 여부, 우선순위

## 3. 명함 스캔 후 연락처 업데이트

### 요구사항

- 명함 이미지를 업로드한다.
- OCR로 이름, 회사, 직책, 전화번호, 이메일을 추출한다.
- 사용자가 추출 결과를 검토하고 연락처로 저장한다.
- 기존 연락처와 이메일 또는 전화번호가 같으면 업데이트 후보로 표시한다.

### 주요 데이터

- 연락처 이름
- 회사명
- 직책
- 전화번호
- 이메일
- 메모
- 원본 명함 이미지 경로

## 4. 문서 사진 촬영 후 스캔본 저장

### 요구사항

- 문서 사진을 업로드한다.
- 스캔본 파일로 저장한다.
- 문서명, 태그, 설명을 입력할 수 있다.
- 스캔본 목록에서 검색하고 상세 보기로 확인한다.

### 주요 데이터

- 문서명
- 파일 경로
- 파일 형식
- 태그
- 설명
- 업로드 일시

## 5. 녹음파일 업로드 후 텍스트 변환 및 보고서 리스트 관리

### 요구사항

- 녹음파일을 업로드한다.
- 음성 인식으로 텍스트를 생성한다.
- 텍스트를 기반으로 보고서 초안을 저장한다.
- 보고서 리스트에서 상태와 내용을 확인한다.

### 주요 데이터

- 녹음파일 경로
- 변환 상태
- 변환 텍스트
- 보고서 제목
- 보고서 요약
- 생성 일시
